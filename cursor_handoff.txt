## 0. 最優先：広告IDの本番切り替え（審査提出前に必ず実施）

### 現状
- `lib/ads.ts` の `USE_TEST_ADS` が **`true` に固定** されている。
- この状態ではすべてのビルド（TestFlight・本番）で **Googleテスト広告** が表示される。
- **Apple/Google審査にテスト広告のまま出すとリジェクトされる可能性がある。**

### 審査提出前にやること
`lib/ads.ts` の以下を変更する：
```
// 変更前（現在）
const USE_TEST_ADS = true;

// 変更後（審査提出用）
const USE_TEST_ADS = typeof __DEV__ !== 'undefined' && __DEV__;
```
- この状態で `eas build` → 審査提出する。
- もし本番ユニットが No fill（AdMob側でまだ配信準備中）でも、審査員の端末では No fill は許容される（広告枠が空でもリジェクト対象ではない）。テスト広告が表示されるほうがリジェクトリスクが高い。

### バージョン情報
- `app.json` の `expo.version` は現在 `1.0.1`。
- 再提出で同バージョンエラーが出たら `1.0.2` に上げること。
- `ios.buildNumber` は `eas.json` の `appVersionSource: remote` + `autoIncrement: true` により自動管理（手動設定不要・削除済み）。

## 1. RedactionEditor（白抜き編集画面）— 完全動作状態

### ファイル
- `components/redaction-editor.tsx`

### ✅ iOS / Android 両方で動作確認済みの機能
- **1本指**: 枠の新規作成・移動・リサイズ（PanGesture）
- **2本指**: ズーム・パン（PinchGesture / PanGesture の Simultaneous）
- **タップ**: 枠の選択（タップで選択 → 削除ボタン表示）
- **削除ボタン**: 枠選択後、枠の右上外側にフローティング表示。`react-native-gesture-handler` の `TouchableOpacity` を使用し、親ジェスチャーとの競合を回避。

### Android ジェスチャー対策（前回セッションで修正済み）
- `Modal` 内に `GestureHandlerRootView` を配置（Android は Modal が別ルート扱いのため必須）。
- ジェスチャー受け口の `Animated.View` に `collapsable={false}` を付与。
- `app/_layout.tsx` 先頭に `import 'react-native-gesture-handler'` を追加。
- 削除ボタンは `GestureDetector` の外（同じ `Animated.View` 配下）に配置し、タッチが親Panに奪われない構造。
- Android DEV 時のみ有効なデバッグログ（`ENABLE_ANDROID_GESTURE_DEBUG`）が全ジェスチャーの onBegin/onUpdate/onEnd に入っている。

### ヒントテキスト
- `「タップ: 選択・削除」` に修正済み（旧: 「長押し: 削除」）。

### 削除ボタン位置
- 極小枠でも表示されるよう、`getDeleteButtonPositionPx()` で枠の右上角外側にピクセル計算し、画面内にクランプして配置。`overflow: 'visible'` + `zIndex/elevation` で隠れ防止。

## 2. 広告（AdMob）アーキテクチャ

### ファイル
- `lib/ads.ts`（SDK初期化・ATT・インタースティシャル）
- `app/(tabs)/index.tsx`（バナー描画・インタースティシャル呼び出し）

### 広告ID管理
- `USE_TEST_ADS` フラグで切り替え。テストID / 本番ID はファイル上部に定義。
- **セクション0 参照**：審査前に必ず本番IDに戻すこと。

### SDK初期化 (`initMobileAds`)
- `isPremium === true` / `isExpoGo()` / 初期化済み のときはスキップ。
- 多重実行防止（`mobileAdsInitInFlight` で排他）。
- iOS の ATT リクエストは `requestTrackingPermission()` を **非同期で並行実行**（SDK初期化をブロックしない）。
- 初期化成功後に `mobileAdsInitialized = true` を設定。

### ATT (App Tracking Transparency)
- `app/_layout.tsx` の `useEffect` で起動後 1200ms + `InteractionManager.runAfterInteractions` 後に `requestTrackingPermission()` を呼ぶ。
- `requestTrackingPermission()` 内で `AppState === 'active'` を待ち、さらに 700ms 遅延後に `requestTrackingPermissionsAsync()` を実行。
- `app.json` の `plugins` に `expo-tracking-transparency` + `userTrackingPermission` を設定済み。
- `ios.infoPlist.NSUserTrackingUsageDescription` も同文言で設定済み。

### バナー (`renderBanner`)
- `adsReady` state が `true` になるまで（= `initMobileAds` 完了まで）描画しない。
- 失敗時は `bannerReloadToken` を 4.5秒後にインクリメントして再マウント（永久非表示にはしない）。
- `isPremium` / `isExpoGo` のときは表示しない。

### インタースティシャル (`showInterstitialThen`)
- 表示前に `initMobileAds(false)` を `await` してSDK初期化を確認。
- `AdEventType.LOADED` → `ad.show()`、`CLOSED` → `done()`、`ERROR` → `done()` + ログ。
- 8秒タイムアウトで `done()` にフォールバック。

## 8. 今後の変更時の注意

- **`USE_TEST_ADS`**: 開発中は `true` でテスト広告を使い、審査提出前に必ず `typeof __DEV__ !== 'undefined' && __DEV__` に戻す。
- **`adsReady` state**: バナー描画は SDK 初期化完了後のみ。この guard を外さないこと。
- **`GestureHandlerRootView`**: Modal 内のジェスチャーには独自の `GestureHandlerRootView` が必須（Android）。
- **削除ボタン**: `GestureDetector` の外に配置 + RNGH の `TouchableOpacity` を使用。RN標準の `TouchableOpacity` に戻すと Android で押せなくなる。
- **ATT呼び出し**: 遅延（1200ms + InteractionManager + 700ms）を短くしすぎるとiPadで表示されない。
